## Robot Framework rules

- Put test cases in the tests folder.
- Group related tests by feature, page, API resource, component, or flow.
- Multiple related tests may be placed in the same .robot file.
- Do not create one large .robot file for unrelated tests.
- Put reusable keywords in resource files.
- Put locators and variables in resource or variables files if the project uses them.
- Keep test cases readable and high-level.
